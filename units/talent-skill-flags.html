Talent skill-type flags

Machine-readable counterpart: units/talent-skill-flags.json. Consumer: web-session/publish-web/search-talent.html (filter tab — web-builder phase).

Flag definitions

Flag CN trigger term Meaning
singleTarget 单体技能 Talent conditions on / modifies single-target skills
multiTarget 群体技能 Talent conditions on / modifies multi-target (AoE) skills
singleHit 单段技能 Talent conditions on / modifies single-hit skills
multiHit 多段技能 Talent conditions on / modifies multi-hit skills

Derivation rules

  1. Explicit type term in CN description → only matching flags set.
  2. Generic skill mention (no type qualifier) → all 4 flags true.
  3. No skill mention → all 4 flags false.
  4. Combined terms (e.g. 群体多段技能) → union of matched flags.
  5. Negation (非多段技能 etc.) → opposite flag set. None found in current data.

Excluded patterns (all-false regardless of generic rule): - 受到技能攻击 / 受到技能伤害 — incoming skill damage on the caster. These describe a defensive reaction to being hit, not the type of skill the caster uses. - 攻击技能 / 治疗技能 — skill purpose (attack vs heal) triggers the generic rule (all-4-true) because no target-count / hit-count type is specified.

Regex patterns

# Incoming-skill exclusion (strip before generic detection):
RE_INCOMING_SKILL = r'受到(?:技能攻击|技能伤害|技能)'

# Specific type patterns:
RE_SINGLE_TARGET        = r'单体技能'
RE_MULTI_TARGET         = r'群体技能'
RE_SINGLE_HIT           = r'单段技能'
RE_MULTI_HIT            = r'多段技能'
RE_MULTI_TARGET_MULTI_HIT = r'群体多段技能'  # sets both mT + mH

# Generic skill (triggers all-4-true after exclusion strip):
Terms: 释放技能 | 技能伤害 | 技能次数 | 技能治疗 | 技能目标 | 技能攻击次数
     | 技能恢复 | 造成技能 | 释放攻击技能 | 释放治疗技能 | 首次释放技能
     | 首回合释放技能 | 施法
